# Decker Tape Products — website redesign

A modern, SEO-first static redesign of [deckertape.com](https://www.deckertape.com/) for
**Decker Tape Products, Inc.** — a pressure-sensitive tape & label manufacturer/converter
in East Hanover, NJ, founded 1969.

**Design direction:** "Everything Tape" — industrial, confident, red/ink/white.
Palette is taken directly from Decker's real logo (`#D1242B` red · `#262223` ink · `#FEFDFD` white).
Signature element is a diagonal **printed-tape strip** (their flagship product) used as the hero
accent and section markers; secondary motif is the **concentric end-of-roll** mark.
Type: Archivo Expanded (display) · Inter (body) · IBM Plex Mono (specs/measurements).

No framework, no build step for the pages themselves — plain HTML/CSS/JS.

## Structure

```
decker-redesign/
├── index.html              Home (hero, products, capabilities, story, CTA)
├── tape.html               Pressure-sensitive tape
├── printed-tape.html       Custom printed tape + tape handles
├── labels.html             Real label categories (special handling, DOT, pictographs…)
├── catalog.html            Full catalog — client-side search/filter over all 1,244 products
├── capabilities.html       Converting floor (printing, slitting, spooling…)
├── about.html              Story + real "Why Decker" copy
├── contact.html            Info + quote form + map
├── shipping.html           Shipping policy (free shipping $99+)
├── returns.html            Returns policy
├── 404.html
├── styles.css              Full design system
├── main.js                 Mobile nav, scroll reveal, quote form, catalog search
├── build.py                Regenerates every inner page from a shared shell
├── catalog.json            1,244 real products (code, name, section) scraped from deckertape.com
├── robots.txt
├── sitemap.xml
└── images/
    ├── decker-logo.svg     Decker's real logo (extracted from their SVG sprite)
    ├── favicon.svg
    ├── roll.svg            Tape-roll hero graphic
    └── og-image.png        1200×630 social share card
```

`index.html` is hand-authored. The inner pages are generated by `build.py`
(shared header/footer/SEO shell) — edit content there and run `python3 build.py` to regenerate.

## Content is verified, not invented

Every factual claim on the site was checked against the live deckertape.com (scraped with `curl`).
Specifically removed because they could **not** be verified on their own site: "1–5 color printing,"
"largest US adhesive-tape printer," "one of two makers of carry handles," fixed business hours, and
a "same-day reply" promise. Kept because they **are** on their site: founded 1969 by George Decker,
"over 55 years," the full capability list, and "the only company in the US to make pre-laminated
tape handles." The catalog (`catalog.json`) is the real product list pulled from their store.

## SEO

- Unique `<title>` + meta description + keywords per page; one `<h1>` per page; semantic landmarks.
- JSON-LD structured data: `Manufacturer` + `LocalBusiness` (NAP) on home,
  `BreadcrumbList` everywhere, `Product` / `ProductGroup` on product pages,
  `AboutPage` and `ContactPage` where relevant.
- Open Graph + Twitter Card meta, canonical URLs, `robots.txt`, `sitemap.xml`.
- Descriptive `alt` text, responsive, keyboard-focus styles, `prefers-reduced-motion` respected.

## What is NOT replicated from the original (and why)

The current deckertape.com runs on **Volusion** (an e-commerce platform). A static site cannot
reproduce the server-backed commerce engine. The following were intentionally **not** rebuilt:

- **Shopping cart & checkout** — needs a commerce backend + payment processor.
- **Customer login / "My Account" / order status / wishlist** — needs auth + a database.
- **Live inventory & pricing** — prices are not shown; the catalog drives **quote requests** instead.

The full **product catalog *is* included** — all 1,244 products (real part numbers, names and lines)
are in `catalog.html` with client-side search and filtering. What's missing is only the *transaction*
(add-to-cart / pay), not the product information. Two pages that were empty on their own site —
**FAQ** ("test test test") and **Leadership Team** (no names) — were left out rather than invented.

The **quote form** on `contact.html` is wired to [Web3Forms](https://web3forms.com) and runs in
"demo mode" until you paste an access key (see **Turn on the quote form** below).

Product **imagery** uses generated SVG/graphics (tape-roll marks, printed-tape mockups), not
Decker's licensed product photos — drop real photos into the marked slots when available.

## Turn on the quote form

The form posts to Web3Forms — works on any static host, no server code.

1. Go to **https://web3forms.com**, enter the email where you want quote requests to land,
   and copy the **access key** they email you (free, no account/password).
2. In `build.py`, replace `YOUR_WEB3FORMS_ACCESS_KEY` with that key, then run `python3 build.py`
   (or edit the same `access_key` value directly in `contact.html`).
3. Submit a test from the live site — it should arrive in your inbox.

Prefer something else? Swap the form's `action` for **Formspree** (`https://formspree.io/f/ID`) or,
if you host on Netlify, add the `netlify` attribute to the `<form>` and drop the access key. The
JS success/error handling works the same.

## Deploy (get it live)

It's plain static files, so any static host works. Easiest first:

**Netlify Drop (no account, ~2 min):** go to **https://app.netlify.com/drop** and drag the
`decker-redesign` folder onto the page. You get a live `*.netlify.app` URL immediately. To use
`deckertape.com`, open Site settings → Domain management → add your domain and point your DNS at
Netlify (they walk you through it). HTTPS is automatic.

**Cloudflare Pages / GitHub Pages / S3+CloudFront** also work — upload the folder contents as the site root.

After deploying, update the absolute URLs (canonical tags, `sitemap.xml`, OG `og:url`/`og:image`)
if the live domain differs from `https://www.deckertape.com/`, and submit `sitemap.xml` in
[Google Search Console](https://search.google.com/search-console) so the new pages get indexed.
